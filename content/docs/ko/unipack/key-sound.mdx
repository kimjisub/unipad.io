`keySound` 파일은 각 버튼에 사운드 파일을 매핑합니다.
한 줄에 하나의 매핑을 작성하며, 공백으로 구분합니다.

## 문법

```
chain  x  y  soundFile  [loop]  [wormhole]
```

| 필드 | Type | 필수 | 설명 |
|------|------|------|------|
| `chain` | int | O | 체인 번호 (1 ~ chain) |
| `x` | int | O | 버튼 X 좌표 (1 ~ buttonX) |
| `y` | int | O | 버튼 Y 좌표 (1 ~ buttonY) |
| `soundFile` | string | O | `sounds/` 폴더 내 파일 경로 |
| `loop` | int | | 반복 설정 (아래 참조) |
| `wormhole` | int | | 재생 후 이동할 체인 번호 |

### loop 값

`loop` 값은 내부적으로 1을 빼서 저장됩니다.

| 작성 값 | 동작 |
|---------|------|
| 생략 또는 `1` | 한 번 재생 (기본) |
| `0` | 무한 반복 |
| `N` (2 이상) | N-1회 반복 |

## 예시

```
1 1 1 kick.wav
1 2 1 snare.wav
1 3 1 hihat.wav 0
2 1 1 bass.wav 1 3
```

- `1 3 1 hihat.wav 0` — 체인 1, (3,1) 위치에 hihat.wav를 무한 반복 재생
- `2 1 1 bass.wav 1 3` — 체인 2, (1,1) 위치에 bass.wav를 한 번 재생 후 체인 3으로 이동

## 순환 큐 (다중 매핑)

같은 `chain x y` 좌표에 여러 사운드를 매핑하면, 버튼을 누를 때마다 **순서대로 순환**하며 재생됩니다.

```
1 1 1 sound_a.wav
1 1 1 sound_b.wav
1 1 1 sound_c.wav
```

(1,1) 버튼을 누를 때마다: sound\_a → sound\_b → sound\_c → sound\_a → ... 순서로 재생됩니다.

<Callout title="유효성 검사">
- 빈 줄과 3개 미만의 컬럼을 가진 줄은 무시됩니다.
- 좌표가 범위를 벗어나면 해당 줄은 건너뛰고 경고가 기록됩니다.
- 참조하는 사운드 파일이 `sounds/` 폴더에 없으면 경고가 기록됩니다.
</Callout>
