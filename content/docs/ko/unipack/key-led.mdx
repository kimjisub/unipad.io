`keyLED` 폴더에는 LED 애니메이션 파일을 저장합니다.
각 파일은 특정 버튼을 눌렀을 때 실행될 LED 시퀀스를 정의합니다.

## 파일명 규칙

파일명 자체가 매핑 정보를 담고 있으며, **공백으로 구분**합니다.

```
chain x y [loop]
```

| 필드 | 설명 |
|------|------|
| `chain` | 체인 번호 (1 ~ chain) |
| `x` | 버튼 X 좌표 (1 ~ buttonX) |
| `y` | 버튼 Y 좌표 (1 ~ buttonY) |
| `loop` | 반복 횟수 (기본: 1, `0` = 무한 반복) |

### 예시 파일명

```
keyLED/
├── 1 1 1          → 체인 1, (1,1), 1회 실행
├── 1 1 1 0        → 체인 1, (1,1), 무한 반복
├── 1 2 3 2        → 체인 1, (2,3), 2회 반복
└── 2 1 1          → 체인 2, (1,1), 1회 실행
```

같은 좌표에 여러 파일을 매핑하면 `keySound`와 마찬가지로 순환 큐로 동작합니다.

## 이벤트 문법

파일 내부에 LED 이벤트를 한 줄에 하나씩 작성합니다. 공백으로 구분합니다.

### `on` — LED 켜기

```
on x y color [velocity]
on x y auto velocity
```

약어: `o`

| 필드 | 설명 |
|------|------|
| `x` | LED X 좌표, 또는 `*` / `mc` (체인 라운드 LED) |
| `y` | LED Y 좌표 |
| `color` | 6자리 HEX 색상 (예: `FF0000`) 또는 `auto` / `a` |
| `velocity` | MIDI velocity (0~127), `auto`일 때 필수 |

#### 색상 지정 방식

| 형식 | 예시 | 동작 |
|------|------|------|
| HEX만 | `on 1 1 FF0000` | 앱에서 빨간색으로 표시 |
| HEX + velocity | `on 1 1 FF0000 5` | 앱에서 빨간색, Launchpad에 velocity 5 전송 |
| auto + velocity | `on 1 1 auto 72` | velocity 72에 매핑된 Launchpad 팔레트 색상 사용 |

#### 특수 좌표

| 토큰 | 의미 |
|------|------|
| `*` 또는 `mc` | Launchpad 상단/측면의 라운드(체인) LED |
| `l` | 로고 LED (지원하지 않음, 무시됨) |

### `off` — LED 끄기

```
off x y
```

약어: `f`. `x` 위치에 `*` / `mc`를 사용할 수 있습니다.

### `delay` — 대기

```
delay ms
```

약어: `d`. 다음 이벤트까지 지정된 밀리초만큼 대기합니다.

### `chain` — 체인 전환

```
chain chainNumber
```

약어: `c`. LED 시퀀스 실행 중 체인을 전환합니다.

## 전체 예시

파일명: `1 3 2 0` (체인 1, 좌표 (3,2), 무한 반복)

```
on 3 2 FF0000
on 3 3 00B8D4
delay 100
off 3 2
off 3 3
delay 50
on * 1 auto 72
delay 100
off * 1
```

<Callout title="유효성 검사">
- 파일명의 좌표가 범위를 벗어나면 해당 파일은 무시됩니다.
- `on` 명령에는 반드시 `color` (4토큰) 또는 `auto` + `velocity` (5토큰)를 지정해야 합니다.
- 인식할 수 없는 명령어가 있으면 해당 줄은 건너뜁니다.
- `loop`이 음수이면 경고가 기록됩니다.
</Callout>
